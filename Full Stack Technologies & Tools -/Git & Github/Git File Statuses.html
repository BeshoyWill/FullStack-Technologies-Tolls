Git File Statuses (Brief Explanation)

In Git, files in your working directory can have different statuses based on whether they've been modified, staged, or committed. These statuses help track changes in your project. Here are the main file statuses:

Untracked

  • The file is new and not yet tracked by Git.
  • It hasn't been added to the staging area with git add.
  • Git doesn't include this file in version control yet.

Unmodified (Clean)

  • The file is being tracked by Git and hasn't been modified since the last commit.
  • No action is needed for this file.

Modified

  • The file has been changed but hasn't been staged yet.
  • Git is aware of the changes, but they won't be included in the next commit unless staged.

Staged

  • The file has been modified and added to the staging area.
  • It is ready to be committed.
  • Changes in this file will be included in the next commit.

Summary of Git File Statuses:

  • Untracked: Not yet added to Git.
  • Unmodified (Clean): No changes since the last commit.
  • Modified: File changed, not staged.
  • Staged: Changes ready to be committed.

الات الملفات في Git (شرح مختصر)

في Git، يمكن أن تكون الملفات في دليل العمل (Working Directory) في حالات مختلفة بناءً على ما إذا كانت قد تم تعديلها أو إعدادها (Staged) أو الالتزام بها (Committed). هذه الحالات تساعدك في تتبع التغييرات في مشروعك. وفيما يلي الحالات الرئيسية للملفات:

غير متعقب (Untracked)

  • الملف جديد ولم يتم تتبعه بواسطة Git بعد.
  • لم تتم إضافته إلى منطقة التحضير (Staging Area) باستخدام git add.
  • Git لا يتعامل مع هذا الملف في التحكم بالإصدارات حتى يتم إضافته.

غير معدل (Unmodified - Clean)

  • الملف يتم تتبعه من قبل Git ولم يتم تعديله منذ آخر التزام.
  • لا حاجة لاتخاذ أي إجراء تجاه هذا الملف.

معدل (Modified)

  • الملف تم تغييره ولكنه لم يُضاف بعد إلى منطقة التحضير.
  • Git على علم بالتغييرات، لكنها لن تُدرج في الالتزام التالي إلا إذا تم إعدادها.

مُحضر (Staged)

  • الملف تم تعديله وأُضيف إلى منطقة التحضير.
  • جاهز لأن يُلتزم به في المستودع.
  • التغييرات في هذا الملف ستُدرج في الالتزام التالي.

ملخص لحالات الملفات في Git:

  • غير متعقب (Untracked): لم تتم إضافته إلى Git بعد.
  • غير معدل (Unmodified - Clean): لا تغييرات منذ آخر التزام.
  • معدل (Modified): الملف تم تغييره لكنه غير مُحضر.
  • مُحضر (Staged): التغييرات جاهزة ليتم الالتزام بها.
