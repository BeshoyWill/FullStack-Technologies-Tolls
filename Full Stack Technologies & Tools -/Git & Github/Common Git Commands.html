Common Git Commands

Common Git Commands (Brief Explanation)

Here are some of the most commonly used Git commands, along with brief explanations:

git init

  • Initializes a new Git repository in your project directory.

#bash

git init

 

git clone

  • Creates a copy of an existing remote Git repository on your local machine.

#bash

git clone <repository-url>

 

git add

  • Adds changes from the working directory to the staging area, preparing them for a commit.
  • To add all changes:

#bash

git add <filename>   ==> to add all files  ==> git add .

 

git commit

  • Saves (commits) the staged changes to the local repository with a message describing the changes.

#bash

git commit -m "Commit message"

 

git status

  • Shows the status of changes in the working directory, including untracked, modified, and staged files.

#bash

git status

 

git log

  • Displays a history of commits in the repository.

#bash

git log

 

git branch

  • Lists all branches in the repository or creates a new branch.
  • To list branches:
  • To create a new branch:

#bash

to list branches :

git branch 

to create a new branch :

git branch <branch-name>

 

git checkout

  • Switches to another branch or a specific commit.
  • To switch to a branch:

#bash

git checkout <branch-name>

 

git merge

  • Combines the changes from one branch into the current branch.

#bash

git merge <branch-name>

 

git pull

  • Fetches and merges changes from a remote repository into your local branch.

#bash

git pull <remote> <branch>

 

git push

  • Sends your local commits to the remote repository.

#bash

git push <remote> <branch>

 

git remote

  • Manages remote repository connections.
  • To add a remote:

#bash

git remote add <name> <url>
